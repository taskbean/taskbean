# Issue tracker: GitHub

Issues and PRDs for this repo live as GitHub issues. Use the GitHub issue workflow for all operations.

## Conventions

- **Create an issue**: create a GitHub issue in `taskbean/taskbean`.
- **Read an issue**: read the issue body, labels, and comments.
- **List issues**: list GitHub issues with appropriate label and state filters.
- **Comment on an issue**: add a GitHub issue comment.
- **Apply / remove labels**: edit GitHub issue labels.
- **Close**: close the GitHub issue with a comment when appropriate.

Infer the repo from `git remote -v` when running inside a clone.

## Pull requests as a triage surface

**PRs as a request surface: no.**

External PRs are not treated as feature requests for `/triage`. Collaborator PRs and external PRs should be handled through normal code-review workflows rather than the issue triage state machine.

## When a skill says "publish to the issue tracker"

Create a GitHub issue.

## When a skill says "fetch the relevant ticket"

Read the matching GitHub issue and include its comments and labels in the working context.
