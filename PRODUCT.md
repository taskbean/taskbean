# Product

## Register

product

## Users

Taskbean is for developers whose daily work includes coding agents. They use it while working locally across repositories, reviewing what agents did, turning tasks into reports, and keeping reminders or follow-ups visible without sending work data to a cloud service.

## Product Purpose

Taskbean makes agent-assisted development work visible and accountable. The CLI gives agents a reliable way to log tasks as they work, while the local desktop PWA gives humans a dashboard for task review, AI-assisted planning, reminders, reports, Chronicle reconciliation, and usage visibility. Success means a developer can understand what was done, what is pending, and what deserves credit or follow-up without scraping transcripts or leaving their machine.

## Brand Personality

Warm, practical, developer-native, technical, and coffee-themed. The interface should feel like a trustworthy local tool with enough personality to be memorable, but never so decorative that it slows down task management or review.

## Anti-references

Avoid generic SaaS-dashboard styling, obvious AI-template composition, overdecorated glassy panels, gratuitous gradients, and cute gamified todo-app patterns. Taskbean should not feel like an enterprise admin console, a decorative marketing mockup, or a terminal-only utility disguised as an app.

## Design Principles

- Local trust first: make privacy, local state, and review boundaries legible without becoming alarmist.
- Receipts over vibes: prioritize concrete task status, evidence, dates, source attribution, and report-ready language.
- Coffee warmth, developer precision: use the coffee identity to add warmth while keeping dense workflows crisp and predictable.
- Human review stays canonical: Chronicle suggestions, usage metadata, and AI output should feel inspectable and reversible.
- Familiar controls, distinctive tone: lean on standard product UI affordances, reserving personality for copy, theming, empty states, and small feedback moments.

## Accessibility & Inclusion

No explicit accessibility target has been set. Future UI work should still preserve readable contrast, keyboard reachability, visible focus states, and reduced-motion alternatives where the existing product already supports them.
